# Drone dashboard

A complete ground-control-station view for a small drone — six components in one screen, all driven by mock data so it works zero-config. Swap the mocks for real `@altara/ros` adapters when you wire it up to your bird.

## What we're building

```
┌──────────────────────────────────────────────────────────────┐
│  ConnectionBar  (full-width status strip)                    │
├─────────────────────────────────────┬────────────────────────┤
│                                     │   ┌──────┐   ┌──────┐  │
│        TimeSeries (Roll/Pitch)      │   │Gauge │   │ Att. │  │
│                                     │   │ Batt │   │      │  │
├─────────────────────────────────────┴────────────────────────┤
│  SignalPanel — alt / spd / hdg / temp / sats / rssi          │
└──────────────────────────────────────────────────────────────┘
```

## Full code

Drop this into a `<Dashboard />` component anywhere in your tree. The only setup is `<AltaraProvider>` at the app root and the one-line stylesheet import.

```tsx
import '@altara/core/styles.css';
import {
  AltaraProvider,
  Attitude,
  ConnectionBar,
  Gauge,
  SignalPanel,
  TimeSeries,
  createMockDataSource,
  randomWalk,
  sineWave,
  stepFunction,
} from '@altara/core';

const altSource = createMockDataSource({ generator: sineWave(0.1, 120), hz: 5 });
const spdSource = createMockDataSource({ generator: sineWave(0.3, 8), hz: 5 });
const hdgSource = createMockDataSource({ generator: sineWave(0.05, 180), hz: 5 });
const tempSource = createMockDataSource({ generator: randomWalk(0.05, 50, 0.5), hz: 5 });
const satsSource = createMockDataSource({ generator: stepFunction(2000, 7, 9), hz: 1 });
const rssiSource = createMockDataSource({ generator: randomWalk(0.05, 120, 0.5), hz: 5 });

export function Dashboard() {
  return (
    <AltaraProvider theme="dark">
      <div
        style={{
          display: 'grid',
          gridTemplateColumns: '1fr 1fr',
          gap: 12,
          padding: 16,
          background: 'var(--vt-bg-base)',
        }}
      >
        <div style={{ gridColumn: '1 / -1' }}>
          <ConnectionBar
            url="ws://localhost:9090"
            status="connected"
            latencyMs={14}
            messagesPerSecond={250}
          />
        </div>

        <TimeSeries
          mockMode
          height={220}
          channels={[
            { key: 'roll', label: 'Roll', unit: '°', color: '#1D9E75' },
            { key: 'pitch', label: 'Pitch', unit: '°', color: '#378ADD' },
          ]}
        />

        <div style={{ display: 'flex', gap: 12, justifyContent: 'space-between' }}>
          <Gauge mockMode min={0} max={100} label="Battery" unit="%" size="md" />
          <Attitude mockMode size={200} />
        </div>

        <SignalPanel
          columns={2}
          signals={[
            { key: 'alt', label: 'Altitude', unit: 'm', dataSource: altSource },
            { key: 'spd', label: 'Speed', unit: 'm/s', dataSource: spdSource },
            { key: 'hdg', label: 'Heading', unit: '°', dataSource: hdgSource },
            { key: 'temp', label: 'Temp', unit: '°C', dataSource: tempSource },
            { key: 'sats', label: 'Satellites', unit: '', dataSource: satsSource },
            { key: 'rssi', label: 'RSSI', unit: 'dBm', dataSource: rssiSource },
          ]}
        />
      </div>
    </AltaraProvider>
  );
}
```

## Wiring to ROS2

Replace each mock source with the matching `@altara/ros` adapter:

| Component | Mock source | ROS2 source | Topic |
|---|---|---|---|
| `Attitude` | `mockMode` | Custom `AltaraDataSource` emitting `roll`/`pitch` channels | `/imu/data` (`sensor_msgs/Imu`) |
| `Gauge` (Battery) | `mockMode` | `createBatteryStateAdapter` | `/battery` (`sensor_msgs/BatteryState`) |
| `TimeSeries` | `mockMode` | `createRosbridgeAdapter`, multi-channel | `/imu/data` (`sensor_msgs/Imu`) |
| `SignalPanel` row "Altitude" | `altSource` | `createNavSatFixAdapter({ axis: 'altitude' })` | `/fix` (`sensor_msgs/NavSatFix`) |
| `SignalPanel` row "Speed" | `spdSource` | `createRosbridgeAdapter` (extract `twist.linear.x`) | `/odometry` (`nav_msgs/Odometry`) |
| `ConnectionBar` | hardcoded `'connected'` | Derive from `useWebSocket` status | n/a |

A minimal real-data swap-in:

```tsx
import { createBatteryStateAdapter } from '@altara/ros';

const battery = createBatteryStateAdapter({
  url: 'ws://localhost:9090',
  topic: '/battery',
});

// then:
<Gauge dataSource={battery} min={0} max={100} label="Battery" unit="%" />
```

## Production checklist

- Replace the hardcoded `<ConnectionBar>` with one driven by `useWebSocket`'s status, latency, and a sliding-window message-rate calculation.
- Add threshold props on the battery `Gauge` (`{ value: 20, color: 'var(--vt-color-warn)' }`, `{ value: 10, color: 'var(--vt-color-danger)' }`).
- For the IMU `TimeSeries` add `thresholds` for the bank-angle limits you care about.
- For ≥500 Hz IMU streams, use `createWorkerDataSource` instead of `createRosbridgeAdapter` — see the [Performance guide](?path=/docs/guides-performance--api-reference).
